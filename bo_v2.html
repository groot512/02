01

인프라 구조 개선 (서버 분리 및 증설)

현 단일 서버 구조의 장애 위험을 해소하고, 회원 증가에 따른 확장이 가능한 안정적 인프라를 확보한다.

1-1

서버 4대 구성

  • 로드밸런서 서버 1대 신규 구축
  • 애플리케이션 서버 2대 신규 구축
  • DB 서버 1대 신규 구축 (기존 서버와 분리 운영)
1-2

애플리케이션 서버 이중화

  • 두 대의 서버에 동일 애플리케이션 구성 배포
  • 현재 운영 중인 2개 애플리케이션 구조 동일 적용
  • 로드밸런서를 통한 트래픽 분산 처리
  • 세션 공유 또는 세션 클러스터링 방안 적용
1-3

DB 서버 분리 운영

  • 애플리케이션 서버와 물리적·논리적 분리
  • 자원 경합 해소 및 성능 안정성 확보
  • DB 이중화는 운영 안정화 이후 단계적 확대 검토
1-4

파일 저장 구조 정비

  • 다중 서버 환경에 적합한 중앙 집중형 파일 저장 방식 도입
  • NAS 또는 오브젝트 스토리지 기반 공유 저장소 구성
  • 서버 간 파일 불일치 방지
1-5

DB 구조 개선 및 쿼리 최적화

  • DB 테이블 구조 점검 및 정규화 수준 검토
  • 주요 쿼리 실행 계획 분석 및 인덱스 최적화
  • 대량 데이터 처리 구간 페이징 및 캐싱 전략 적용
1-6

모니터링 및 장애 대응 체계 구축

  • 서버·DB·애플리케이션 실시간 모니터링 시스템 구축
  • 장애 발생 시 관리자 알림 체계 (SMS/이메일) 구축
  • 주요 서비스 장애 대응 매뉴얼 수립
02

개인정보 보호 강화

회원 정보 보호 수준을 상향하여 보안 사고를 예방하고, 개인정보 보호법에 부합하는 안전한 서비스 환경을 구축한다.

2-1

주요 개인정보 컬럼 암호화 적용

  • 주민등록번호, 연락처, 주소, 계좌정보 등 암호화 저장
  • AES-256 등 국가정보원 권고 기준 알고리즘 적용
  • 암호화 키 관리 체계 수립
  • 기존 데이터 암호화 마이그레이션 수행
2-2

관리자 계정 보안 강화

  • 비밀번호 복잡도 정책 적용 (대소문자, 숫자, 특수문자 조합)
  • 정기 비밀번호 변경 및 강제 변경 로직 구현
  • 로그인 실패 횟수 제한 및 계정 잠금 정책
  • 관리자 로그인 이력 기록·관리
03

백업 체계 정비

시스템 장애 및 데이터 손실에 대비하여 안정적인 복구 체계를 마련하고, 서비스 연속성을 보장한다.

3-1

DB 및 시스템 정기 백업 정책 수립

  • 일 단위 증분 백업 + 주 단위 전체 백업 정책 수립
  • 백업 보관 기간 설정 (일 백업 7일, 주 백업 4주)
  • 백업 정책 문서 작성 및 관리
3-2

DB 및 파일 백업 시스템 구축

  • DB 네이티브 백업 또는 전용 백업 도구 활용 구현
  • 파일 백업 원격 저장소 자동 전송 구성
  • 자동 백업 스케줄링 체계 적용
  • 백업 완료·실패 알림 체계 연동
3-3

복구 절차 정비

  • 장애 유형별 복구 절차 문서화
  • 분기별 1회 이상 복구 테스트 수행
  • RTO/RPO 목표 설정 및 정량 관리
04

상생지원센터 및 가맹점 시스템 개선

권한 체계를 명확히 하고 내부 통제 구조를 정비하여 운영 안정성과 보안 수준을 강화한다.

4-1

시스템 로직 재설계

  • 상생지원센터·가맹점·관리자 권한 구분 명확화
  • 역할 기반 접근제어(RBAC) 구조 정비
  • CRUD 기능별 동작 체계 정리
  • 수기 입력 체계 개선 및 자동화 전환
  • CMS 적용 시 가맹 시스템 연계 구조 정비
4-2

보훈마켓 전용 통합 CMS 구축

  • 임시 CMS를 보훈마켓 전용 관리 시스템으로 전환 개발
  • 기능 구조 재설계 및 관리 효율성 강화
4-3

기능별 접근 권한 세분화

  • 메뉴 및 기능 단위 권한 설정 기능 개선
  • 개인정보 및 정산 관련 기능 접근 통제 강화
4-4

권한 변경 이력 관리

  • 권한 부여·변경·회수 이력 기록
  • 감사 목적의 로그 관리 체계 마련
05

가입 절차를 간소화하고 처리 시간을 단축하여 사용자 만족도를 향상시키며, 행정 처리 효율성을 높인다.

가입 절차 간소화

OCR 기반 자동 입력 도입

  • 증빙자료 촬영 시 OCR 자동 정보 추출
  • 수기 입력 최소화를 통한 가입 시간 단축
  • 인식 오류 보완 기능 제공
  • 서버 기반 정보 검증 로직 추가
5-3

반려 대상자 재가입 절차 개선

  • 자동 반려 처리 로직 정비
  • 반려 대상 항목만 수정 가능하도록 개선
5-4

안정성 강화

  • 회원 증가 대비 처리 안정성 확보
  • 가입 대상이 아닌 회원 정리 및 관리 체계 정비
  • 오류 처리 및 안내 체계 개선
5-5

약관 및 개인정보 동의 체계 정비

  • 개인정보 수집·이용 동의 항목 정비
  • 선택·필수 항목 구분 명확화
  • 약관 버전 관리 기능 도입

인프라 및 보안 기반 구축

서비스의 안정성과 보안성을 확보하는 기반 작업으로서 최우선 추진한다. 서버 분리·이중화, DB 분리, 파일 저장 구조 정비, 개인정보 암호화, 관리자 보안 강화, 백업 체계 구축, 모니터링 시스템 구축을 포함한다.
